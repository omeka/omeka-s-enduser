# Omeka S User Manual

Omeka S is a web publication system. It is used by universities, galleries, libraries, archives, and museums. Omeka S creates a local network of independently curated exhibits. These exhibits share a pool of items and their metadata.

The sections of this manual match the navigation on the left-hand side of the [administrative dashboard](admin-dashboard.md): 

**Sites**  
Creating, editing, and managing sites. Sites are the public facing parts of an Omeka S installation.

**Resources**  
Creating, editing, and managing the content. This includes items, item sets, media, vocabularies, and resource templates.

**Admin**  
Access to functionality in the administrative interface varies by user role. It can include: users, jobs, assets, and settings.

**Modules**  
This section opens with basic information on managing the modules that extends Omeka S's functionality. It then offers specific documentation for the modules created by the Omeka Team. These are listed in alphabetical order.

This manual also includes a [glossary](glossary.md), instructions on conducting [searches](search.md), and a guide on how to turn on [error logging](errorLogging.md).

**Contributing to the user manual**
You can find information on how to contribute changes, ask questions, and lodge issues on [the Github repository for this user manual](https://github.com/omeka/omeka-s-enduser){target=_blank}.

We use external technology, MkDocs, to generate our user manual. We are working to make both the manual and our language and images within it more accessible. Please get in contact if you have any problems, or suggestions to improve the accessibility of this documentation.
